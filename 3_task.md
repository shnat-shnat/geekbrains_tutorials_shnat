# Введение в контроль версий
Контроль версий используется для отслеживания изменений исходного кода, а также для командной работы над кодом. 
Система контроля версий позволяет возвращаться к различным версиям проектаю

**Основные команды системы контроля версий git:**

git init - для того, чтобы указать папку, в которой  мы будем отслеживать изменений;
git status - для просмотра текущего состояния;
git add - для добавления содержимого для последующего коммита;
git commit - для фиксации изменений;
git log - для просмотра журнала изменений;
git branch - список веток, также для создания новой ветки;
git checkout - для переключения между ветками;
git diff - для определения разницы между версиями;
git clone - для того, чтобы клонировать репозиторий;
git pull - для того, чтобы отправить информацию из удаленного репозитория на свой репозиторий и объединить ветки;
git push - чтобы выложить информацию из своего репозитория на удаленный репозиторий.
